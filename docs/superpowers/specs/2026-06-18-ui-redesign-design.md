# Дизайн: редизайн интерфейса NetSearch («понятно с первого взгляда»)

**Дата:** 2026-06-18
**Статус:** утверждён, готов к планированию реализации

## 1. Цель

Сделать интерфейс NetSearch удобнее и красивее, чтобы рядовому сотруднику было
сразу понятно, что и как делать. Текущий вид функционален, но «сухой»: мелкие
кнопки в ряд, нет подсказок, пустой экран без объяснений, статус только текстом,
а фильтр по дате в UI вообще не подключён.

Это редизайн **UI-слоя** существующего WPF-приложения. Логика ядра
(`NetSearch.Core`) и поведение индексации/поиска не меняются — за одним
исключением: фильтр по дате будет наконец проброшен из UI в поиск.

## 2. Зафиксированные решения

- **Визуальный стиль:** светлый дружелюбный. Светлый фон, белые «поверхности»,
  синий акцент `#2D7FF9`, мягкие границы, скруглённые кнопки с состояниями
  hover/disabled.
- **Удобство (всё включено):** подсказки (tooltips) везде + плашка «как начать»;
  полоса прогресса при индексации; удобные фильтры (календарь дат, подсказки
  размера, быстрые кнопки типов); яркое главное действие + иконки.

## 3. Подход к темизации

Всё оформление выносится в централизованный ресурс-словарь
`src/NetSearch.App/Themes/AppTheme.xaml` (цвета-кисти, типографика, стили
`Button` — обычная/акцентная/вторичная, `TextBox`, `ToggleButton`, `ComboBox`,
`DataGrid`, `StatusBar`, `Expander`). Подключается в `App.xaml` через
`Application.Resources` → `MergedDictionaries`. Никаких разрозненных инлайновых
стилей — единый вид и простая правка.

Размер окна ≈ 1060×680.

### Блоки и роли

- **Шапка:** название + «Обновить» (акцентная цветная кнопка с иконкой) и
  «Настройки» (вторичная).
- **Hero-поиск:** крупное поле с иконкой-лупой и плейсхолдером «Введите имя
  файла или папки…».
- **Режим поиска:** три сегмент-кнопки `Имя / Маска / Regex` (radio-поведение)
  вместо выпадающего списка. Рядом «По содержимому» и тумблер «Фильтры».
- **Панель фильтров:** раскрывающаяся, скрыта по умолчанию.
- **Таблица результатов:** стилизованная, подсветка строки под курсором; поверх
  пустой таблицы — онбординг-плашка.
- **Низ:** индетерминированная полоса прогресса со счётчиком/папкой и «Отмена»
  (видна при индексации) + строка статуса.

## 5. Поведение

### Фильтры

- **Даты:** два `DatePicker` для «изменён от/до». Подключаются новые свойства
  ViewModel `ModifiedAfter`/`ModifiedBefore` (тип `DateTimeOffset?`), которые
  пробрасываются в `QueryBuilder.Build(...)` вместо нынешних жёстких `null`.
- **Размер:** два поля; формат подсказан плейсхолдером («напр. 10МБ»); парсинг
  (КБ/МБ/ГБ) уже есть в `QueryBuilder.ParseSize`.
- **Быстрые типы:** кнопки `PDF`, `Word`, `Excel`, `Фото`, подставляющие
  расширения в поле «Тип». Команда `QuickType(string group)` во ViewModel +
  ручное поле рядом. Маппинг групп:
  - PDF → `pdf`
  - Word → `doc, docx`
  - Excel → `xls, xlsx`
  - Фото → `jpg, jpeg, png, gif, bmp`
- **Только файлы / только папки:** два чекбокса, маппятся на существующий
  `EntryKind`: только «файлы» отмечен → `FilesOnly`; только «папки» → `FoldersOnly`;
  оба отмечены или оба сняты → `All`.

### Онбординг и подсказки

- **Плашка «Как начать»:** поверх таблицы, видна когда индекс пуст или не задан
  ни один путь (свойство `ShowOnboarding`). Три нумерованных шага:
  1. Настройки → добавьте сетевой путь
  2. Обновить → дождитесь индексации
  3. Начните печатать имя — результат сразу
- **Tooltips** простым языком на каждом контроле (поиск, режимы, «По
  содержимому» = «ищет текст внутри уже найденных файлов», «Обновить» =
  «пересканировать сетевые пути», фильтры).

### Прогресс

Индетерминированная `ProgressBar` снизу, видна при `IsBusy`; рядом текст «идёт
индексация… N · текущая папка» (течёт через существующий прогресс-колбэк
`CrawlProgress`) и кнопка «Отмена». По завершении — статус «обновлён в HH:MM».

## 6. Окно «Настройки»

Приводится к той же теме: пояснение под полем путей («Например: `\\HOST-01\work`
или `Z:\` — по одному на строку»), tooltips, аккуратные отступы, кнопки в общем
стиле. Логика сохранения не меняется.

## 7. Технические добавления (изолированные)

- `src/NetSearch.App/Themes/AppTheme.xaml` — ресурсы и стили; мерджится в
  `App.xaml`.
- Конвертеры: `EnumMatchConverter` (сегмент-режим, `value == parameter`);
  встроенный `BooleanToVisibilityConverter` для панелей/плашек.
- `PlaceholderHelper` — attached property для плейсхолдеров (переиспользуемый).
- ViewModel (`MainViewModel`): добавляются `ModifiedAfter`/`ModifiedBefore`
  (`DateTimeOffset?`), `FiltersExpanded` (bool), `ShowOnboarding` (bool,
  производное от пустого индекса / отсутствия путей), `FilesOnly`/`FoldersOnly`
  (bool-чекбоксы, чьи изменения пересчитывают `SelectedKind`), команда
  `QuickType`. Логика
  поиска/индексации не меняется — только UI-слой и проброс дат.

## 8. Тестирование и поставка

- UI проверяется сборкой (0 ошибок) + ручным smoke. Логика ядра (41 тест) не
  трогается и должна остаться зелёной.
- При выносе маппинга «быстрый тип → расширения» в чистый хелпер — добавить
  мелкий юнит-тест на маппинг.
- В завершение — пересборка `publish/NetSearch.exe`.

## 9. Что НЕ входит (YAGNI)

- Тёмная тема / переключатель тем.
- Локализация (интерфейс на русском).
- Изменение алгоритмов индексации/поиска (кроме проброса фильтра по дате).
